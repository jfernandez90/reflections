How is the staging area different from the working directory and the repository? What value do you think it offers?
	The staging area is a temporary intermediate "space" in which you add specific files prior to making them a new commit.  This allows you an easier way to group related files and separate those that are not related.
How can you use the staging area to make sure you have one commit per logical change?
	You can add single files at at time, if they are not related, or you can add multiple files to the staging area so that when you commit related files are saved
